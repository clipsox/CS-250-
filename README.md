# CS 250 Module Eight Journal

Throughout this course, I learned that understanding user needs is one of the most important parts of developing successful software. Instead of immediately thinking about how to build a solution, I learned to first understand what problem the user is trying to solve. Creating user stories helped me accomplish this because they described the system from the user's perspective and focused on the value each feature would provide. Breaking requirements into user stories also made it easier to prioritize work, communicate with the team, and verify that each completed feature met the customer's expectations. I now see user stories as a valuable tool for translating business requirements into manageable development tasks.

This course also changed the way I approach developing software. Before learning Scrum and Agile, I tended to think of software development as completing one large project from beginning to end. Now I understand the value of building software in small, manageable increments while continuously gathering feedback and adapting to change. In future projects, I plan to use Agile practices such as maintaining a prioritized product backlog, creating detailed user stories, planning work through sprints, conducting sprint reviews, and participating in retrospectives to identify opportunities for improvement. I also appreciate the importance of collaboration throughout the entire development process instead of waiting until the end of a project to identify problems.

To me, being a good team member in software development means communicating openly, collaborating with others, and being willing to both give and receive constructive feedback. Every member of a Scrum team has a different responsibility, but everyone shares the same goal of delivering value to the customer. Throughout this course, I learned that successful teams depend on trust, accountability, and clear communication just as much as technical ability. As I continue developing my programming and software engineering skills, I plan to carry these Agile principles into future academic, personal, and professional projects because they encourage continuous improvement while keeping the user's needs at the center of development.
